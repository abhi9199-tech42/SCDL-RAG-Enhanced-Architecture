Multi-Signal Detection

NLI contradiction scoring + lexical overlap + entity matching. Chi-parameter combines all signals into one verdict.

Plain English Explanations

Every verdict includes a human-readable explanation of which signal flagged the contradiction and why.

Auto-Improvement Loop

When mu < 0.1, reads the chi breakdown, searches for better evidence, and recomputes. Fixes 3 out of 4 hallucinations.

Batch Endpoint

Collapses N serial HTTP calls into 2. 6.8x throughput improvement. 0 errors tested up to 500 cases per batch.

Async Queue

Fire-and-forget with configurable worker pool (default 4 threads). 55 tasks/s throughput. 0 failures at 5000 task backlog.

Enterprise Ready

Bearer token auth, connection pooling (HTTP/2, 100 keepalive), zero degradation over 30-min sustained load. Air-gapped deployment.

API

Six endpoints for detection, batch processing, fixing, async jobs, and health checks.

POST /v1/detect Detect hallucination in a single QA pair against evidence documents
POST /v1/detect/batch Batch detect up to 500 cases in one request
POST /v1/detect-and-fix Detect + auto-repair by searching a document pool
POST /v1/batch/detect-and-fix Batch version of detect-and-fix
POST /v1/async/detect Queue a detection job, returns task_id immediately
GET /v1/async/result/{task_id} Poll for async job result
GET /v1/health Health check with uptime and model status

Get Started

Install the Python SDK or use the REST API directly. Runs on your infrastructure.

# Install
pip install scdl-rag

# Use
from scdl_rag import SCDLRAG

client = SCDLRAG(api_key="sk_...")
result = client.detect(
    question="What is the capital of France?",
    answer="The capital is London.",
    documents=[{"id": "d1", "content": "Paris is the capital of France."}]
)
print(result.hallucination_detected)  # True
print(result.mu_score)                # 0.42
print(result.explanation)             # "NLI: contradiction (0.92)"

import { SCDLRAG } from "scdl-rag";
